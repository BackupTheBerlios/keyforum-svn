FAQ:

->            Cos'è KeyForum ?

E' un forum senza un server centrale, orientato al P2P, dove ogni client connesso contribuisce
alla diffusione e alla memorizzazione dei messaggi. E' un forum anonimo, perchè non è possibile
associare un messaggio all'indirizzo IP da cui è stato generato e, girando in locale, è più 
veloce e robusto di un'analoga soluzione basata su server centralizzato.

->            Come funziona ?

Si esegue il programma e ci si può già connettere al forum con il proprio browser preferito, 
si possono registrare utenti e mandare messaggi. E' possibile collegarsi a diverse board 
esistenti, basate su KeyForum, oppure crearne ed amministrarne di proprie. Appena si scrive
un messaggio KeyForum si occupa di spedirlo agli altri nodi connessi appartenenti alla stessa 
board, che lo riceveranno immediatamente se sono on-line, o in seguito se sono off-line. 
Algoritmi specializzati fanno si che tutti i client connessi abbiano l'intera base messaggi 
sul proprio disco fisso nel minor tempo possibile.

->            E la sicurezza ?

KeyForum è basato sulle chiavi pubbliche/private RSA per garantire l'univocità di utenti e 
messaggi. Dal momento che ogni client rispedisce senza modifiche ogni messaggio che riceve, 
non è possibile sapere se un messaggio è stato generato da chi me lo ha effettivamente inviato.

->            E se sono un utente FASTWEB o sono dietro una LAN o un FIREWALL ?

Potrai comunque utilizzare KeyForum ricevendo e spedendo messaggi agli altri client.

->            Con quali sistemi operativi funziona ?

Essendo scritto in PERL e in PHP ed utilizzando componenti standard come Apache e MySQL 
KeyForum può funzionare virtualmente su qualunque sistema operativo in cui siano disponibili. 
Al momento sono previste distribuzioni per Windows e Linux

->            E' gratuito ? E' disponibile il sorgente ?

Si, KeyForum sarà sempre un prodotto gratuito ed è rilasciato sotto licenza GPL. Il programma 
viene eseguito direttamente dal sorgente, quindi anche la risposta alla seconda domanda è 
certamente si

->            E se ho altre domande ?

Puoi farle nel Forum di supporto http://www.keyforum.net/forum

FAQ:
->            What is KeyForum ? 
KeyForum is a serverless forum system,
based on the P2P concept, in which every client connected to the network 
shares its copy of the forum database with other clients. It is anonymous, since 
it is impossible to link a message to the IP address it originated from. 
Moreover, since the forum runs locally on each user's PC, it is more robust and 
faster than a conventional server-based forum system.

->            How does it work? 
Once the program is running, the board is fully functional: 
you can register, log in and browse through the board using your preferred 
browser, reading messages and posting new ones. You can subscribe to existing 
KeyForum-based boards created by others, or create new boards and share them 
with anyone. The different boards are not related to each other and the client 
forwards to and collects messages only from subscribed boards. Specialized 
algorithms ensure that every client connected to a board gets a complete copy of 
its database in the shortest time.

->            What about security? 
KeyForum relies on the public/private RSA key model to 
identify and guaranteee the authenticity of registered users and messages. Since 
every client shares the messages it gets without modifying them, it is 
impossible to know if a message originated from or was simply forwarded by the 
IP address it comes from.

->            What if I am behind a FIREWALL? 
There is no problem using KeyForum behind a 
firewall.

->            On which operating systems does KeyForum run? 
KeyForum is written in Perl and 
PHP and uses standard components like Apache and MySql, which makes it virtually 
possible to run it on any operating system which supports them. So far, it is 
being tested on both Linux and Windows.

->            Is it FREE? Is the project OPENSOURCE? 
Yes, KeyForum is free and OpenSource, distributed under the GPL license.
